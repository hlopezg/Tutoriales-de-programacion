Layouts

Los Layouts son una parte fundamental en Android ya que me permite organizar todos los controles visuales de nuestra aplicación en cada pantalla que mostremos, podríamos decir que un Layout es como una ventana dentro de una aplicación de escritorio, pero tiene varias diferencias. Por ejemplo si queremos hacer un formulario con campos de texto y botones, todo eso debe estar alineado dentro de un Layout que envuelve todo e incluso puede haber varios Layout dentro deotros (que es lo más comun) para poder dar un orden adecuado.

Los Layout disponibles son los siguientes:

･ Linear Layout

･ Relative Layout

･ Table Layout

･ Absolute Layout

･ Frame Layout

En este curso solo hablaremos de los LinearLayout y RelativeLayout ya que son los más utilizados. Antes utilizaba los TableLayout para cuando quería mostrar datos dentro de una tabla pero con el tiempo me di cuenta que puedo hacer lo mismo utilizando LinearLayout (uno por cada fila) o en algunos casos con RelativeLayout.

Podemos incluir Layout dentro de Layouts o Views dentro de Layout, pero hay que tratar de evitar de hacer esto (o de crear la menor cantidad de Layout posibles) ya que esto afecta en la memoria del dispositivo.

LinearLayout

Es el más simple de utilizar (prácticamente es el unico que yo utilizo) todos los controles que pondremos dentro de un LinearLayout se van colocando uno detrás de otro, y podemos indicar se irán organizando de manera vertical u horizontal.

Para Realizaremos un pequeño formulario con 1 Layout y 3 controles (un TextView, EditText y Button), podemos realizarlo tanto por código como por el apartado gráfico.



<?xml version="1.0" encoding="utf-8"?>

<LinearLayout
  xmlns:android= "http://schemas.android.com/apk/res/android"
  xmlns:tools="http://schemas.android.com/tools"
  android:layout_width="match_parent"
  android:layout_height="match_parent"
  android:orientation="vertical"
  >
	
  <TextView
	android:layout_width="wrap_content"
	android:layout_height="wrap_content"
	android:text="@string/hello_world"
	/>
	
  <EditText
	android:id="@+id/editText_tutorial_1"
	android:layout_width="wrap_content"
	android:layout_height="wrap_content"
	android:layout_weight="1"
	android:ems="10">
	<requestFocus/>
	</EditText>
	
  <Button
	android:id="@+id/button_tutorial_1"
	android:layout_width="wrap_content"
	android:layout_height="wrap_content"
	android:text="Button"
	/>
</LinearLayout>

Es decir que nuestros Layout y controles tienen la siguiente estructura:


layout_width y layout_height

Si nos fijamos en la parte gráfica del código anterior, podemos ver que el que el botón que creamos tiene el ancho según el largo del texto que contiene, si queremos que tome el ancho completo del dispositivo tenemos que cambiar la siguiente propiedad android:layout_width="wrap_content" por android:layout_width="match_parent" De esta manera indicamos que el botón tome el ancho máximo que puede tomar el Layout en donde se incluye, quedando nuestro apartado gráfico esta manera.


Un error común que solemos cometer y asignar malos valores a nuestro layout_width y layout_height. Haremos algo malo a propósito para darnos cuenta que debemos comprender bien estas propiedades, vamos a cambiar el alto del Layout interno (el que contienen el TextView y el EditText) y le daremos el tamaño del primer Layout. Vemos que el primer botón toma en ancho del LinearLayout en el cual está contenido, por lo tanto los otros 2 botones no se logran mostrar por que no caben en la pantalla. Entonces si queremos que los 3 botones se muestren y aprovechen todo el ancho de la pantalla, tendremos que aprender sobre el atributo weight. La propiedad Weight nos permita dar un "peso" a nuestros controles, lo que vamos a hacer es darle un peso igual a cada uno de nuestros botones agregando la siguiente línea a cada uno android:layout_weight="1" para que quede de la siguiente manera. < Button android:id="@+id/button1" android:layout_width="0dip" android:layout_height="wrap_content" android:text="Button 1" android:layout_weight="1"/> El valor para el layout_width de los tres botones lo hemos definido con 0dip para que no tome en cuenta esta propiedad y sí la del peso y pueda distribuir equitativamente los componentes. Ademas al LinearLayout que contiene los botones le agregamos la siguiente línea android:weightSum="3" Esto nos dice cuanto va a ser el peso total que van a contener todos los controles del Layout, como en este caso son 3 botones nuestro peso de 1 cada uno, en total se sumara un peso de 3, es decir que se distribuirá el peso entre los 3 botones de manera equitativa ocupando un ancho de un 33% cada uno.

Por ultimo vamos a aprender a alinear los controles, esto se logra con la propiedad gravity en este caso alinearemos el botón a la derecha y tendremos otro botón a la izquierda. Para esto el LinearLayout principal lo dejaremos con orientación horizontal, luego crearemos un botón como cualquier otro que se obicará al lado izquierdo y el botón para el lado derecho lo dejaremos dentro de un LinearLayout agregándole el atributo android:gravity="right", con esto hacemos que todo el contenido del Layout se mueva a la derecha.

RelativeLayout Los RelativeLayout son ideales para utilizarlos cuando queremos poner un objeto hacia un lado de otro (arriba, abajo, izquierda, derecha), es decir para posicionar un objeto en relación a otro objeto, pero también podemos posicionarlo con respecto al área que corresponde al RelativeLayout. En algunos caos utilizar RelativeLayout permite ahorrarnos el trabajo de tener un LinerLayout dentro de otro, de esta manera optimizamos memoria. 
Propiedades
• android:layout_above indica que el elemento se posicionará justo arriba del elemento que tiene el ID definido como valor de este atributo.
• android:layout_toLeftOf indica que el elemento se posicionará a la izquierda del elemento cuyo ID coincida con el definido en el valor de este atributo.
• android:layout_toRightOf posiciona al elemento a la derecha del elemento cuyo ID coincida con el proporcionado en el valor de este atributo.
• android:layout_bottom posiciona al elemento debajo del que tenga la ID proporcionada en el valor del atributo.

Siempre debemos indicar la ID del control (View) del cual queremos basar la posición, por ejemplo: android:layout_toLeftOf="@id/button1" dónde el elemento cuyo ID es button1 se definiá en el archivo XML.

También podemos posicionar nuestro View de acuerdo al contenedor (RelativeLayout) en el que se encuentra, con los siguientes atributos cuyos valores solo pueden ser true o false.
• android:layout_alignParentTop Indica al View que su borde superior deberá estar alineado con el borde superior del RelativeLayout.
• android:layout_alignParentBottom Indica al View que su borde inferior deberá estar alineado con el borde inferior del RelativeLayout.
• android:layout_alignParentLeft Indica al View que su borde izquierdo deberá estar alineado con el borde izquierdo del RelativeLayout.
• android:layout_alignParentRight Indica al View que su borde derecho deberá estar alineado con el borde derecho del RelativeLayout.
• android:layout_centerHorizontal Permite centrar horizontalmente al View con respecto a su RelativeLayout.
• android:layout_centerVertical Permite centrar verticalmente al View con respecto a su RelativeLayout.
Ejemplo Cuando creamos un proyecto el .xml creado por defecto viene de esta manera. 

< RelativeLayout  
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:paddingBottom="@dimen/activity_vertical_margin"
    android:paddingLeft="@dimen/activity_horizontal_margin"
    android:paddingRight="@dimen/activity_horizontal_margin"
    android:paddingTop="@dimen/activity_vertical_margin"
    tools:context=".MainActivity" >

</RelativeLayout>

Cuando al EditText le doy el valor android:layout_toRightOf="@+id/textView1" Significa que lo estoy alineando al lado derecho del control con la ID "textView1" (el cual es un TextView), a su vez al Button cuando le doy el valor android:layout_below="@+id/editText1" significa que lo estoy posicionando debajo de EditText y también le agrego el valor android:layout_alignParentRight="true" que posiciona al botón apegado al lado derecho del RelativeLayout.
